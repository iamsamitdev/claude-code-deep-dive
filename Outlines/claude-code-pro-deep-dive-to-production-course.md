# Claude Code มือโปร: ดำน้ำลึกสู่ Production

#### 15 ชั่วโมง (ราคาโปรโมชั่นเปิดตัว เริ่มต้น "ฟรี")

> หลักสูตรอบรมออนไลน์ (สอนสด) — สถาบันไอทีจีเนียส เอ็นจิเนียริ่ง

---

ทุกวันนี้นักพัฒนาเกือบทุกคนรู้จัก "Vibe Coding" และเคยให้ AI ช่วยเขียนโค้ดมาบ้างแล้ว แต่ความจริงที่หลายคนยังไม่รู้คือ — Claude Code ไม่ได้เป็นเพียงเครื่องมือเติมโค้ดอัตโนมัติ (Autocomplete) แต่เป็น **Agentic Coding Tool** ที่ทำงานบน Terminal คิดเป็นขั้นตอน อ่านโค้ดทั้งโปรเจกต์ แก้หลายไฟล์พร้อมกัน รันคำสั่ง ทดสอบ และแม้แต่ส่งมอบงานขึ้น Server ได้ด้วยตัวเอง ปัญหาคือนักพัฒนาส่วนใหญ่ใช้มันได้เพียง 10% ของความสามารถจริง — สั่งงานทีละบรรทัด ไม่เคยตั้งค่า Project Memory ไม่รู้จัก Sub-agent ไม่เคยต่อ MCP และไม่รู้ว่าจะทำให้ทั้งทีมใช้ AI ร่วมกันอย่างเป็นมาตรฐานได้อย่างไร

หลักสูตรนี้ออกแบบมาเพื่อพา "ดำน้ำลึก" ลงไปถึงแก่นของ Claude Code โดยใช้โปรเจกต์จริง — **"ระบบคลังสินค้าเบิกจ่าย"** ที่พัฒนาด้วย Next.js (App Router), Prisma และ PostgreSQL — เป็นสนามฝึก ผู้เรียนจะได้เรียนรู้ตั้งแต่การวาง `CLAUDE.md` ให้เป็นสมองของโปรเจกต์ การบริหาร Context การสร้าง Custom Slash Commands การออกแบบ **Sub-agents** เฉพาะทาง (เช่น Code Reviewer, Test Writer, Security Auditor) การต่อ MCP และ Hooks ไปจนถึงการทำให้ทั้งทีมใช้ Claude Code ร่วมกันผ่าน Git แล้วปิดท้ายด้วยการ Containerize ด้วย Docker วาง CI/CD ด้วย GitHub Actions และ Deploy ขึ้น VPS Ubuntu จริงพร้อม Nginx และ HTTPS โดยมี Claude Code เป็นคู่หูตลอดเส้นทางจาก Vibe สู่ Production

***หมายเหตุ*** หลักสูตรนี้เป็นการสอนสดออนไลน์ผ่านโปรแกรม Zoom Meeting และมีการบันทึกวิดีโอการอบรมส่งให้ทบทวนย้อนหลังทั้งหลักสูตร ในส่วนของการ Deploy หลักสูตรนี้ใช้ **VPS แบบทั่วไป** ที่ติดตั้ง Ubuntu Linux ผู้เรียนสามารถเลือกใช้ผู้ให้บริการ VPS รายใดก็ได้ (ทั้งในและต่างประเทศ) เนื้อหาและขั้นตอนทั้งหมดใช้ได้กับ VPS มาตรฐานทุกราย ไม่ผูกกับผู้ให้บริการเฉพาะเจ้าใดเจ้าหนึ่ง ผู้เรียนควรมีบัญชี Claude ที่ใช้งาน Claude Code ได้ (Pro/Max หรือ API)

## ภาพรวมหัวข้อการฝึกอบรม

- แนวคิด Agentic Coding และการใช้ Claude Code อย่างมือโปร — ต่างจากการใช้ AI เติมโค้ดทั่วไปอย่างไร
- การวาง `CLAUDE.md` ให้เป็น Project Memory และการบริหาร Context Window อย่างมีประสิทธิภาพ
- การสร้าง Custom Slash Commands และการใช้ Plan Mode เพื่อออกแบบก่อนลงมือ
- การออกแบบและใช้งาน **Sub-agents** เฉพาะทางสำหรับ Review, Test และ Security
- การต่อ Claude Code กับเครื่องมือภายนอกด้วย **MCP (Model Context Protocol)** และการทำ Automation ด้วย **Hooks**
- การทำงานเป็นทีมด้วย Claude Code — แชร์ Config ผ่าน Git, Code Review และ PR Workflow
- การ Containerize ด้วย Docker, วาง CI/CD ด้วย GitHub Actions และ Deploy ขึ้น VPS Ubuntu จริงพร้อม Nginx + HTTPS

**สวัสดีครับ ผมอาจารย์สามิตร โกยม** — ผู้สอนหลักสูตรนี้ครับ ผมจบการศึกษาระดับปริญญาโทด้านเทคโนโลยีสารสนเทศ จากมหาวิทยาลัยเทคโนโลยีพระจอมเกล้าพระนครเหนือ และอยู่ในวงการพัฒนาโปรแกรมและไอทีมากว่า 15 ปี

- **ประสบการณ์ทำงาน:** เคยเป็นโปรแกรมเมอร์ในบริษัทเอกชน, ที่ปรึกษางานไอทีให้ธุรกิจ SME หลายแห่ง และเป็นอาจารย์พิเศษสอนวิชาคอมพิวเตอร์ที่มหาวิทยาลัยเทคโนโลยีพระจอมเกล้าพระนครเหนือ (KMUTNB)
- **ประสบการณ์สอน:** ปัจจุบันเป็น CEO และผู้ก่อตั้งสถาบัน IT Genius Engineering ได้ถ่ายทอดความรู้ด้านการเขียนโปรแกรมและพัฒนาเว็บให้ผู้เรียนมาแล้วหลายพันคน ตั้งแต่ระดับนักศึกษาไปจนถึงระดับองค์กร
- **ความเชี่ยวชาญ:** สอนหลักสูตรด้าน Web Development, DevOps และเทคโนโลยีสมัยใหม่มากมาย ทั้ง Next.js, Docker, CI/CD ตลอดจนการประยุกต์ใช้ AI Tools และ Agentic Coding ในงานพัฒนาจริง
- **ความมุ่งมั่น:** เป้าหมายของผมคือทำให้การเรียนรู้เทคโนโลยีที่ทันสมัยเป็นเรื่องง่ายและเป็นไปได้สำหรับทุกคน ผู้เรียนจะได้รับการถ่ายทอดความรู้ด้วยวิธีที่เข้าใจง่าย เป็นกันเอง และนำไปใช้ได้จริง

## ผู้เรียนต้องมีพื้นฐานอะไรบ้าง?

- มีพื้นฐานการเขียนเว็บด้วย JavaScript / TypeScript หรือ React มาบ้าง (ไม่จำเป็นต้องเชี่ยวชาญ)
- เข้าใจแนวคิดของฐานข้อมูลเชิงสัมพันธ์ (Relational Database) เบื้องต้น
- เคยใช้งานคำสั่งพื้นฐานบน Command Line / Terminal และ Git
- ไม่จำเป็นต้องเคยใช้ Claude Code, Sub-agent, Docker, CI/CD หรือ Linux Server มาก่อน — หลักสูตรปูพื้นฐานให้ครบ
- มีคอมพิวเตอร์ส่วนตัวสำหรับใช้ในการอบรม (Windows, macOS หรือ Linux) และบัญชี Claude ที่ใช้ Claude Code ได้

## จุดเด่นของหลักสูตร

- **เจาะลึก Claude Code ระดับมือโปร** — ไม่ใช่แค่ "สั่งให้เขียนโค้ด" แต่เข้าใจ Agentic Loop, Context Management, Sub-agent, MCP และ Hooks ที่ทำให้ใช้ AI ได้เต็มประสิทธิภาพจริง
- **มี Sub-agent เป็นหัวใจของหลักสูตร** — อุทิศทั้งวันให้กับการออกแบบและใช้ Sub-agent เฉพาะทาง ซึ่งเป็นทักษะที่หาเรียนตามบทความฟรีได้ยากมาก
- **ออกแบบมาเพื่อการทำงานเป็นทีมโดยเฉพาะ** — สอนการแชร์ `CLAUDE.md`, Slash Commands และ Sub-agent ผ่าน Git ให้ทั้งทีมใช้ AI อย่างเป็นมาตรฐานเดียวกัน
- **เรียนผ่านโปรเจกต์เดียวจบ** — พัฒนาระบบคลังสินค้าเบิกจ่ายด้วย Next.js ตั้งแต่บรรทัดแรกจนแอปออนไลน์จริง เห็นภาพรวมทั้งกระบวนการ Vibe → Production
- **ครอบคลุมตั้งแต่ Dev จนถึง Production** — ปิดช่องว่างที่บทเรียนทั่วไปมักหยุดอยู่แค่ localhost ด้วยการพาไปจนถึง Docker, CI/CD และการตั้งค่า VPS จริง

## เนื้อหาการอบรม

### **Day 1:** ปูพื้น Claude Code ระดับมือโปร และตั้งต้นระบบคลังสินค้าด้วย Next.js

- **ทำความรู้จัก Claude Code เชิงลึก:** เข้าใจแนวคิด Agentic Coding ที่ทำงานบน Terminal ความต่างจาก AI เติมโค้ดทั่วไป และ Agentic Loop (อ่าน-วางแผน-แก้-รัน-ตรวจสอบ) ที่เป็นหัวใจของการทำงาน
- **ติดตั้งและตั้งค่าให้พร้อมใช้งานจริง:** การติดตั้ง Claude Code, การเลือกใช้โมเดล (Opus / Sonnet / Haiku) ให้เหมาะกับงาน และการตั้งค่า Permission Mode อย่างปลอดภัย
- **Vibe Coding อย่างมีวินัย:** เทคนิคการเขียน Prompt ที่ดี การวาง Context ให้ AI เข้าใจเป้าหมาย และการใช้ Plan Mode เพื่อให้ Claude ออกแบบก่อนลงมือเขียน
- **`CLAUDE.md` — สมองของโปรเจกต์:** เขียน Project Memory ให้ Claude เข้าใจ Tech Stack, Coding Conventions และกฎเฉพาะของทีม (เช่น โครงสร้างโฟลเดอร์ การตั้งชื่อ การจัดรูปแบบโค้ด)
- **Workshop:** ใช้ Claude Code สร้างโปรเจกต์ Next.js (App Router) ของระบบคลังสินค้า ออกแบบ Prisma Schema (`Product`, `StockTransaction`) เชื่อมต่อ PostgreSQL และวาง `CLAUDE.md` ฉบับแรกของโปรเจกต์

### **Day 2:** พัฒนาฟีเจอร์คลังสินค้าให้ใช้งานได้จริง พร้อมเทคนิค Context และ Slash Commands

- **บริหาร Context Window อย่างมือโปร:** การใช้ `/clear`, `/compact`, การ Reference ไฟล์ด้วย `@` และหลักการ "Context Economy" เพื่อให้ Claude ทำงานแม่นยำในโปรเจกต์ขนาดใหญ่
- **Custom Slash Commands:** สร้างคำสั่งของตัวเอง (เช่น `/add-feature`, `/create-crud`) เพื่อมาตรฐานงานซ้ำ ๆ และทำให้คำสั่งกลายเป็นสินทรัพย์ของทีม
- **พัฒนาฟีเจอร์หลักของระบบ:** สร้าง CRUD จัดการสินค้า, ฟังก์ชันรับเข้า (Stock In) และเบิกจ่าย (Stock Out) ด้วย Server Actions / Route Handlers พร้อม Validation และตรรกะป้องกันการเบิกเกินจำนวน
- **สร้างหน้า Dashboard และรายงานสต็อก:** แสดงยอดคงเหลือ ประวัติการเคลื่อนไหว และการแจ้งเตือนสินค้าใกล้หมด โดยให้ Claude ช่วยออกแบบ UI และ Query
- **Workshop:** เดินหน้าพัฒนาระบบคลังสินค้าให้ครบฟีเจอร์ พร้อมฝึก Debug และ Refactor ร่วมกับ Claude Code อย่างเป็นระบบ

### **Day 3:** ดำน้ำลึก Sub-agents, MCP และ Hooks — หัวใจของหลักสูตร

- **Sub-agents คืออะไร ทำไมจึงเปลี่ยนเกม:** เข้าใจแนวคิด Context Isolation และเหตุผลที่ Agent เฉพาะทางทำงานได้ดีกว่า Agent ตัวเดียวที่ทำทุกอย่าง
- **ออกแบบและสร้าง Sub-agent เฉพาะทาง:** สร้าง `code-reviewer`, `test-writer` และ `security-auditor` พร้อมเทคนิคเขียน System Prompt, การกำหนด Tools ที่ใช้ได้ และการเลือกโมเดลให้เหมาะกับบทบาท
- **Orchestration และการทำงานแบบขนาน:** การมอบหมายงานให้หลาย Sub-agent ทำงานร่วมกัน และการเรียก Agent ซ้อนกันเพื่อจัดการงานที่ซับซ้อน
- **ต่อโลกภายนอกด้วย MCP (Model Context Protocol):** เชื่อม Claude Code เข้ากับเครื่องมือภายนอก เช่น ฐานข้อมูล, GitHub และบริการอื่น ๆ เพื่อให้ AI เข้าถึงข้อมูลจริงได้
- **Automate ด้วย Hooks:** ตั้งค่า Hooks ให้รัน Lint, Format หรือ Test โดยอัตโนมัติในจังหวะที่กำหนด
- **Workshop:** สร้าง Sub-agent สำหรับ Review โค้ดและเขียน Test ให้ระบบคลังสินค้า ต่อ MCP เข้ากับฐานข้อมูล และตั้ง Hook ตรวจคุณภาพโค้ดก่อน Commit

### **Day 4:** ทำงานเป็นทีมด้วย Claude Code, Docker Container และ CI Pipeline

- **มาตรฐานการใช้ AI ร่วมกันในทีม:** การแชร์ `CLAUDE.md`, Slash Commands และ Sub-agent ผ่านโฟลเดอร์ `.claude/` บน Git เพื่อให้ทุกคนในทีมใช้ AI ไปในทิศทางเดียวกัน
- **Git Workflow ร่วมกับ Claude Code:** การให้ Claude ช่วยเขียน Commit Message, สร้าง Pull Request และทำ Code Review บน GitHub อย่างเป็นระบบ
- **พื้นฐาน Docker ที่นักพัฒนาต้องรู้:** ความสัมพันธ์ของ Image, Container และ Registry พร้อมเขียน `Dockerfile` สำหรับ Next.js แบบ Multi-stage Build และ Standalone Output ให้ได้ Image ขนาดเล็กและปลอดภัย โดยมี Claude Code ช่วยร่าง
- **จัดการหลาย Service ด้วย Docker Compose:** รวม Next.js และ PostgreSQL เข้าด้วยกัน พร้อมจัดการ Environment Variables, Volume และ Network
- **สร้าง CI Pipeline ด้วย GitHub Actions:** เขียน Workflow สำหรับ Build และ Test อัตโนมัติ พร้อม Push Docker Image ขึ้น Container Registry และการจัดการ Secret อย่างปลอดภัย
- **Workshop:** Containerize ระบบคลังสินค้า, จัดการด้วย Docker Compose และวาง CI Pipeline ให้ Build/Test/Push อัตโนมัติเมื่อมีการ Push โค้ด

### **Day 5:** Deploy สู่ Production จริงด้วย VPS Ubuntu, Nginx และ CD อัตโนมัติ

- **เตรียมความพร้อม VPS Ubuntu Linux:** การเชื่อมต่อผ่าน SSH, การสร้าง User สำหรับใช้งาน, การตั้งค่า Firewall (UFW) และ Security Hardening เบื้องต้น โดยให้ Claude Code ช่วยร่างสคริปต์ตั้งค่า
- **ติดตั้ง Docker บน Server และนำระบบขึ้นใช้งาน:** การ Pull Docker Image จาก Registry และรันระบบทั้งหมดด้วย Docker Compose บนสภาพแวดล้อม Production
- **ตั้งค่า Nginx เป็น Reverse Proxy:** การ Config `proxy_pass`, การจัดการ Header, Gzip และ Timeout ให้เหมาะกับการทำงานของ Next.js
- **ติดตั้งโดเมนและ HTTPS:** การชี้โดเมนมายัง Server และติดตั้งใบรับรอง SSL ฟรีด้วย Let's Encrypt (Certbot) เพื่อให้เว็บทำงานบน HTTPS
- **ทำ CD อัตโนมัติและดูแลระบบด้วย Claude Code:** ต่อยอด GitHub Actions ให้ Deploy ขึ้น VPS โดยอัตโนมัติเมื่อมีการ Push โค้ด พร้อมแนวทางการดู Log, Backup, Rollback และการใช้ Claude Code ช่วยวิเคราะห์ปัญหา (Incident Response) ในระบบจริง
- **Workshop:** Deploy ระบบคลังสินค้าขึ้น Production เต็มรูปแบบ ตั้งค่า CD อัตโนมัติ และสรุป Best Practices การทำงานกับ AI Agent ในทีมระยะยาว
